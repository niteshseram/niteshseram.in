@AGENTS.md

# Project overview

Personal portfolio and blog at niteshseram.in. Built with Next.js 16 (App Router), React 19, TypeScript (strict), and Tailwind CSS v4. Blog posts are authored in MDX and processed by fumadocs-mdx. Uses `motion` for animation, `matter-js` for physics, `cmdk` for the command menu, and `@base-ui/react` for headless primitives.

# Commands

- `pnpm dev` — start dev server
- `pnpm build` — production build
- `pnpm lint` / `pnpm lint:fix` — lint with oxlint
- `pnpm fmt` / `pnpm fmt:check` — format with oxfmt
- `pnpm tc` — typecheck with `tsgo --noEmit`

# Architecture

## Routing

Next.js App Router. `params` is a `Promise` in Next.js 16 — always `await params` before accessing fields.

Route groups:

- `(app)` — main site pages (`/`, `/about`, `/writing`, `/writing/[slug]`). Shares a layout with `Navbar` and `Footer`.
- `(llms)` — machine-readable endpoints: `llms.txt`, `llms-full.txt`, `about.md`, `experience.md`, `projects.md`, `speaking.md`, and `/writing/[slug].mdx`. All served via `route.ts` handlers using `cacheLife('max')`.

Other top-level routes:

- `/design` — internal design-system reference (tokens, typography, Button/Anchor/Tooltip demos).
- `/api/search` — fumadocs-core search index for writing posts.
- `robots.ts`, `sitemap.ts`, `not-found.tsx`, `icon.svg` live directly under `src/app/`.

## Content pipeline

MDX files live in `src/content/writing/`. Fumadocs-mdx processes them via `source.config.ts` and generates types into `.source/`. The source loader in `src/lib/source.ts` exposes `writingSource`. Post helpers in `src/lib/writing.ts` handle fetching, sorting, and filtering (drafts excluded when `VERCEL_ENV === 'production'`).

LLM-friendly post text is generated by `src/lib/get-llm-text.ts` (strips frontmatter, injects title/summary/date/tags header). Aggregate sections (about, experience, projects, speaking) for `llms-full.txt` are built in `src/lib/llms-content.ts` from the static data files.

### MDX frontmatter schema

```yaml
title: string (required)
summary: string (required)
publishedAt: date string (required, coerced to Date)
tags: string[] (default [])
draft: boolean (default false)
# readingTime is auto-calculated — do not set manually
```

Code blocks use Shiki with `catppuccin-latte` (light) and `catppuccin-mocha` (dark).

## Styling

Tailwind CSS v4 with `@tailwindcss/postcss`. Theme colors use oklch() via CSS custom properties defined in `src/app/globals.css` (`:root` for light, `.dark` for dark). The dark variant is configured as `@custom-variant dark (&:where(.dark, .dark *))`.

### Design tokens (color names)

`background`, `foreground`, `card`, `card-foreground`, `muted`, `muted-foreground`, `surface`, `surface-foreground`, `popover`, `popover-foreground`, `border`, `input`, `ring`, `brand`, `brand-foreground`, `brand-muted`, `link`, `code`, `code-bar`, `code-foreground`, `code-number`, `code-highlight`, `selection`, `selection-foreground`

### Fonts

Configured in `src/lib/fonts.ts` via `next/font/google`:

- `--font-sans` (DM Sans, 300/400/500) — body text
- `--font-serif` (Instrument Serif, 400 + italic) — headings
- `--font-mono` (JetBrains Mono, 400) — code

### Class grouping (see AGENTS.md for full rules)

Group Tailwind classes by purpose in `cn()`: Layout > Spacing > Shape > Typography > Colors > Interaction > Transitions > Hover/focus > Data/aria. One string per group, no inline comments.

## Components

Components live in `src/components/` organized by domain:

- `ui/` — primitives: `anchor`, `button`, `tooltip`, `codeblock`, `command`, `contribution-graph`, `falling-stack`.
- `home/` — homepage sections: `hero`, `tech-stack-section`, `projects-section`, `writing-section`, `experience-section`, `speaking-section`, `github-contribution/`, `contact-section`, plus the shared `section-heading`.
- `about/` — `about-intro`, `off-the-clock`, `timeline-section`.
- `writing/` — post-rendering chrome: `post-header`, `post-meta`, `post-footer`, `post-actions`, `post-row`, `tag-chip`, `prose`, `mdx-components`.
- `command-menu/` — ⌘K command menu (`cmdk` + `@base-ui/react` dialog): `index`, `trigger`, `dialog`, `groups`, `search`.
- Top-level shared: `navbar`, `footer`, `logo`, `theme-provider`, `theme-switcher`, `terminal` (typing animation), `scroll-progress-rail`.

Key patterns:

- Props are typed with `Readonly<{...}>`.
- `cn()` (clsx + tailwind-merge) for class composition — defined in `src/lib/utils.ts`.
- `cva()` (class-variance-authority) for variant-driven components (e.g., `Anchor`).
- `'use client'` only on components that need browser APIs or hooks. Server components are the default.
- Icons come from `react-icons/pi` (Phosphor Icons).
- Animations use `motion/react` (the `motion` package, not `framer-motion`).
- Reusable hooks live in `src/utils/`: `use-copy-button`, `use-global-shortcut`.

### Button

`src/components/ui/button.tsx` — renders as `<button>` or `<Anchor>` depending on whether `href` is passed. Variants: `brand`, `outline`, `ghost`. Sizes: `xs` through `xl`. Supports icon-only mode via `isLabelHidden`, leading/trailing icons via `addonPosition`, pill shape via `borderRadius="full"`, and optional `tooltip` / `tooltipSide` / `tooltipAlign`.

### Anchor

`src/components/ui/anchor.tsx` — renders Next.js `<Link>` for internal URLs, plain `<a>` for external. Variants: `default`, `brand`, `unstyled`. Weights: `inherit`, `medium` (default), `normal`. Automatically adds `rel="noreferrer noopener"` and `target="_blank"` for external links.

### Tooltip

`src/components/ui/tooltip.tsx` — thin wrapper around `@base-ui/react/tooltip` exposing `TooltipProvider`, `Tooltip`, `TooltipTrigger`, `TooltipContent`.

## SEO & metadata

- `pageMetadata()` in `src/lib/metadata.ts` generates consistent `Metadata` objects.
- JSON-LD structured data via `src/lib/jsonld.ts` (Person, WebSite, Blog, BlogPosting). Person/WebSite graph is injected in the root layout; BlogPosting is injected per post.
- Dynamic OG images generated per blog post at `opengraph-image.tsx` (fonts loaded via `src/lib/og-font.ts`).
- Site constants in `src/config/site.ts` (`SITE_URL`, `AUTHOR`, `SITE_DESCRIPTION`, `SITE_TAGLINE`, `WRITING`, `SITE_LANGUAGE`).
- Google Analytics injected via `@next/third-parties/google` only when `NEXT_PUBLIC_GA_ID` is set and `VERCEL_ENV === 'production'`.

## Data & config

Static data exports in `src/data/`: `about.ts`, `projects.ts`, `experience.ts`, `tech-stack.tsx`, `talks.ts`, `social-links.ts`, `timeline.ts`. Plain TypeScript — no database.

Config in `src/config/`: `site.ts` (site constants/author), `nav.ts` (nav items with icons and command-menu keywords), `theme.ts` (theme metadata for the switcher).

# Code style

## Language & tooling

- TypeScript strict mode. No `any` — use proper types (`typescript/no-explicit-any` is `error`).
- oxlint for linting, oxfmt for formatting. No ESLint or Prettier.
- oxfmt config: single quotes, 2-space indent, 80-char width, semicolons, trailing commas, sorted imports.
- Prefer `toSorted()`, `findIndex()`, and other modern array methods (don't mutate with `.sort()`).
- Use `Readonly<{...}>` for component prop types (local `Props` type is the convention).
- Use `!= null` / `== null` for nullish checks — covers both `null` and `undefined` in one check. Don't rely on truthy checks when `0` or `''` are valid values.
- Use `const` by default; `no-var` and `prefer-const` are errors.

## React & components

- **Named exports for components.** Only Next.js files that require a default export (`page.tsx`, `layout.tsx`, `route.ts`, `opengraph-image.tsx`, `not-found.tsx`, `sitemap.ts`, `robots.ts`) use `export default`. Everything in `src/components/` exports by name.
- **Plain function declarations** — no `React.FC`, no `FunctionComponent`, no top-level arrow-function components.
- **File names are kebab-case** (`post-header.tsx`, `falling-stack.tsx`, `scroll-progress-rail.tsx`). Component identifiers are PascalCase.
- **`'use client'` is the first line** of any client file, above imports.
- **Server components are the default.** Only reach for a client component when you need browser APIs, hooks, event handlers, or a client-only library. Prefer splitting a client leaf off a server parent over making a whole tree client.
- **JSX props and prop-type members are ordered alphabetically** (including `aria-*`). Event handlers live in their alphabetical slot, not at the end.
- **Dynamic routes must define `generateStaticParams`.** Server-rendered text endpoints wrap expensive work in `'use cache'` + `cacheLife(...)` from `next/cache`.

## Accessibility

- Decorative icons and ornamental elements get `aria-hidden="true"`.
- Icon-only buttons (`isLabelHidden`) require a meaningful `label` — the `Button` component renders it into `sr-only` automatically. Also set `aria-label` when the visible label would be ambiguous.
- Use semantic landmarks over generic `<div>`: `<section aria-label="...">` / `<article>` / `<nav aria-label="...">` / `<ol>` / `<ul>`. Non-primary `<nav>`s still need an `aria-label`.
- `jsx-a11y` rules are enabled as warnings — fix them rather than ignoring.

## Animation & styling

- Avoid the `motion-safe:` prefix — the user's system has Reduce Motion enabled, so those animations never play for them. Either animate unconditionally (when safe) or gate on explicit opt-in.
- Use `motion/react` (the `motion` package), not `framer-motion`.
- Class grouping in `cn()` follows AGENTS.md — one string per purpose group, no inline comments.
